Convert JSON subfiles to a YAML document presentation.

This helper can be used to convert an IOP-JSON document into an IOP-YAML one. IOP-JSON only handles includes as "presentation data". This function can be used to convert those includes into a YAML document presentation, so that the repacked YAML data will keep the same includes and subfiles.

If type IOP struct definition that is referenced by the IOP path is provided with st, the generated presentation will use proper includes: includeraw for strings, and include for the rest.

This function allow to unpack an IOP structure encoded in YAML format in one call. This is equivalent to:

This function cannot be used to unpack a class; use t_iop_yunpack_ptr_ps instead.

Parameters

[out]outPointer on the IOP structure to write.

Convert IOP-YAML to an IOP C structure using the t_pool().

This function acts exactly as t_iop_yunpack_ps but allocates (or reallocates) the destination structure.

This function MUST be used to unpack a class instead of t_iop_yunpack_ps, because the size of a class is not known before unpacking it (this could be a child).
