IOP_COPY_SHALLOW 

Perform a shallow copy instead of a default deep copy.

Only the root structure fields are copied when using this flag. By default, all fields of the structure are copied recursively.

Do not perform reallocation of the output value on copy.

For class instances, the output object must be initialized and the original __vptr of the output object is not modified.

Like IOP_COPY_MULTIPLE_ALLOC, the memory pool must be a by-frame memory pool if IOP_COPY_SHALLOW is not used.

This flag is not available for iop_dup functions.

Cancel the allocation if the dup size is bigger than MEM_ALLOC_MAX.

In that case, the returned pointer will be set to NULL. This option allows to apply dedicated protections so we can handle the error instead of crashing when IOPs get too big.
