Connect to an SCTP endpoint.

This function begins to establish a connection with the remote. The connection is not established at the end of the function. When it will be established, the on_connect_cb function will be called with its status (successfully connected or failed to connect).

The returned pointer will be necessary to send messages and/or close the connection.

Parameters
[in]source_addrsthe IPs used as source, can be empty
[in]dest_addrsthe destination IPs, multiple IPs means multi-homing
[in]portthe port to connect to on the remote end
[in]sndbufmaximum size of sending buffer (man 7 socket, SO_SNDBUF)
[in]entity_idthe name of the remote endpoint (see sctp_conn_t)
[in]privuser context that will be accessible from callbacks
[in]on_connect_cbfunction called on connection success/failure
[in]on_data_cbfunction called when data is received
[in]on_disconnect_cbfunction called when connection ends

Listen to incoming SCTP connections.

This function will return a listening context.

The returned context needs to be deallocated eventually.

The purpose of the returrned context is to listen for new connections. It doesn't allow the user to send messages and the disconnection callback will not be called for this context.

Parameters
[in]addrsaddresses to listen to, multiple IPs means multi-homing
[in]portthe port to listen to
[in]sndbufmaximum size of sending buffer (man 7 socket, SO_SNDBUF)
[in]entity_idthe name of the remote endpoint (see sctp_conn_t)
