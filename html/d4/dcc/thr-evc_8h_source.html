24/*
25 * An eventcount usage pattern is usually of the form:
26 *
27 * #thread 1
28 *
29 * for (;;) {
30 * if (some_check) {
31 * again:
32 * do_stuff;
33 * } else {
34 * uint64_t key = thr_ec_get(ec);
35 *
36 * if (some_check)
37 * goto again;
38 * thr_ec_wait(ec, key);
39 * }
40 * }
41 *
42 *
43 * #thread 2
44 *
45 * make "some_check" become true;
46 * thr_ec_signal(ec)
47 *
48 *
49 * This allows a pthread_cond_t kind of usage without the overhead of
50 * acquiring a mutex, but this also means that the "thing" we want to wait
51 * upon is also updated/checked without holding a mutex else it makes no sense
52 * to use an eventcount.
53 *
54 * Note that this is usually a good idea to be sure that the structure which
55 * uses the eventcount is aligned to the cache lines to avoid false sharing.
56 * This isn't enforced as usually there is other data read/written in the same
57 * fashion than the eventcount so we let the eventcount user share the
58 * cacheline for this.
59 *
60 * thr_ec_get(ec):
61 * get the current eventcount "key". This performs a memory barrier which
62 * isn't cheap, so do it only when what you're protecting just failed (see
63 * above)
64 *
65 * thr_ec_wait(ec, key):
66 * Wait on the ec if its current key really is "key" (else it just quits),
67 * until we're signalled. This really works like futex(ec, FUTEX_WAIT, key)
68 * in a way (and indeed is a wrapper around futex(2)), so read futex(2) for
69 * more details.
70 *
71 *
72 * thr_ec_signal_n(ec, n):
73 * wake up at most "n" threads currently waiting on the eventcount. This
74 * API is usually not called directly, see below.
75 *
76 * thr_ec_signal_relaxed(ec):
77 * thr_ec_signal(ec):
78 * wake up at most one thread currently waiting on the eventcount.
79 * The relaxed version doesn't signal if there is no waiter seen (in a racy
80 * way) at signal time.
81 * The strict version always performs one atomic increment before it checks
82 * for waiters, but is non racy wrt proper use of thr_ec_wait.
83 *
84 * thr_ec_broadcast_relaxed(ec):
85 * thr_ec_broadcast(ec):
86 * wake up all the threads currently waiting on the eventcount.
87 * See thr_ec_signal/thr_ec_signal_relaxed for the discussion about the
88 * relaxed version.
89 */
