19/* IOP Channels with internal event loop.
20 *
21 * When using languages or systems that uses an internal event loop, it is
22 * required, in order to use IOP Channels, to start an Intersec event loop in
23 * a thread.
24 *
25 * This can be tricky to do it right. This header and its source file provides
26 * high-level functions to do it.
27 *
28 * ic_el_module_init() must be called before using any other functions.
29 * ic_el_module_stop() and ic_el_module_cleanup() must be called at the end to
30 * stop the internal thread and resources.
31 *
32 * Internally, the functions are protected of by an internal lock.
33 *
34 * WARNING: Unless specified, all functions *MUST* be called with external
35 * locks released to avoid dead-lock.
36 * For example, in Python, the GIL must be released before calling the
37 * functions.
38 *
39 * The callbacks are called with the internal lock released.
40 * WARNING: The callbacks are called in the internal thread, and not in the
41 * thread of the caller.
42 */
43

333 * if the query has been successful with \p status.
