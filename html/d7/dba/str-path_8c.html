◆ path_relative_to()

int path_relative_to ( char buf[static PATH_MAX],
const char * from,
const char * to )

Create a relative path from one path to another path.

from and to can be absolute or relative paths from the current working directory. If from ends with a '/', it is treated as a directory and the created relative path startpoint is within the directory.

Examples:

  • The relative path from "/a/b/c/d" to "/a/b/e/f" is "../e/f".
  • The relative path from "/a/b/c/d/" to "/a/b/e/f" is "../../e/f".
  • The relative path from "a/b/c" to "d/e/" is "../../d/e".
  • The relative path from "a/b/c/" to "a/b/c" is "c".
  • The relative path from "/a/b/c" to "d/e" is "../d/e" with the current working directory set to "/a".
Parameters
[out]bufbuffer where the path will be written.
[in]lenlength of the buffer.
[in]fromthe path that defines the startpoint of the relative path.
[in]tothe path that defines the endpoint of the relative path.
Returns
-1 if the length of the absolute path of from or to, or the length of the resulting path exceed PATH_MAX. Length of the resulting path otherwise.

◆ path_va_extend()

int path_va_extend ( char buf[static PATH_MAX],
const char * prefix,
const char * fmt,
va_list args )

Extend a relative path from a prefix. 

prefix + fmt = prefix + / + fmt

A '/' will be added between /p prefix and /p fmt if necessary.

If /p fmt begins with a '/', the prefix will be ignored. If /p fmt begins with '~/', the prefix will be ignored, and ~ replaced by the HOME prefix of the current user.

Parameters
[out]bufbuffer where the path will be written.
[in]prefix
[in]fmtformat of the suffix
[in]argsva_list containing the arguments for the fmt formatting
Returns
-1 if the path overflows the buffer, length of the resulting path otherwise.
