Create a new memory ring-pool.

The memory ring-pool is an allocator working with frames of memory which rotates on a extending ring.

This is quite a fifo-pool with the oldest frames always re-appended at the head of the pool.

Parameters

◆ mem_ring_reset()

void mem_ring_reset ( mem_pool_t * _rp)

Force clean the ring pool from empty memory blocks.

Adapted from mem_stack_pool_reset().

Only keeps the current frame so as the one that fits the best the current needs regarding the mean allocation size (see code for details).

Should be called when the pool is idle: won't do anything if called while there are still some frames not released.

XXX The clean-up is supposed to be automatically triggered every 64K frame releases, using this function to force it means that you have a clear understanding of what you are doing.

◆ mem_ring_rewind()

void mem_ring_rewind ( mem_pool_t * _rp,
const void * ckpoint )

Rewind the ring-pool at a given checkpoint.

The rewind procedure will release all the framed created between the current active frame and the checkpoint. Then it will unseal the frame which was sealed when mem_ring_checkpoint() was called.

Be careful that it will not restore the frames released after the checkpoint.

The checkpoint cannot be reused after this call.
