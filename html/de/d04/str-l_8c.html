◆ lstr_ascii_iequal()

bool lstr_ascii_iequal ( const lstr_t s1,
const lstr_t s2 )

Returns whether s1 and s2 contents are case-insentively equal.

This function should only be used in case you have a small number of comparison to perform. If you need to perform a lot of checks with the exact same string, you should first lowercase (or uppercase) both string and use the case-sensitive equality that will be much more efficient.

This function is not unicode-aware.

◆ lstr_dlevenshtein()

int lstr_dlevenshtein ( const lstr_t s1,
const lstr_t s2,
int max_dist )

Returns the Damerau–Levenshtein distance between two strings.

This is the number of additions, deletions, substitutions, and transpositions needed to transform a string into another. This can be used to detect possible misspellings in texts written by humans.

Parameters
[in]s1the first string to compare.
[in]s2the second string to compare.
[in]max_distthe distance computation will be aborted if it is detected that the result will be strictly greater than this limit, allowing to save CPU time; can be -1 for no limit.
Returns
-1 if max_dist was reached, the Damerau–Levenshtein distance between s1 and s2 otherwise.

void lstr_transfer_sb ( lstr_t * dst,
struct sb_t * sb,
bool keep_pool )

copies src into dst transferring memory ownershipt to dst

The src is a string buffer that will get reinitilized by the operation as it loses ownership to the buffer.

If keep_pool is false, the function ensures the memory will be allocated on the heap (sb_detach). If keep_pool is true, the memory will be transfered as-is including the allocation pool.

lstr_t lstr_trim_pkcs7_padding ( lstr_t padded)

Trim 1 to 8 padding bytes (PKCS#7).

This implementation is not fully PKCS#7 compliant: only the last padding byte is read, the other padding bytes are not checked.

◆ mp_lstr_dup_utf8_reversed()

lstr_t mp_lstr_dup_utf8_reversed ( mem_pool_t * mp,
const lstr_t v )

Duplicates v on the mem_pool and reverse its content.

This function reverse character by character, which means that the result contains the same characters in the reversed order but each character is preserved in it's origin byte-wise order. This guarantees that both the source and the destination are valid utf8 strings.

In case of error, LSTR_NULL_V is returned.
