Set the URL of the server exposing the APIs.

The route will be added to the server url in the final document. The exposed URL will be: <server_url>/route So for example, with the server url being "http://my.server.com:1337", the exposed URL will be: http://my.server.com:1337/route

◆ t_iop_openapi_whitelist_rpc()

void t_iop_openapi_whitelist_rpc ( iop_openapi_t *nonnull openapi,
const lstr_t fullname )

Whitelist an RPC in the IOP OpenaAPI application.

Only RPCs that have been whitelisted will be exposed in the OpenAPI description. The RPC must be written in the format "<iface_fullname>.<rpc_name>".

Warning
If this function is never called (so the whitelist is empty), all RPCs will be exposed.

Create a new IOP OpenAPI application.

This object can be used to add types, RPCs, interfaces to the application. Then, a YAML OpenAPI description of the application can be generated.

Parameters
[in]titleThe title of the application
[in]versionThe version the application, must be a semver string.
[in]moduleThe IOP module used for the OpenAPI application.
[in]routeThe route name. Every RPCs will be exposed in the route /<route>/<iface_alias>/<rpc_name>.
Returns
An IOP OpenAPI application.
