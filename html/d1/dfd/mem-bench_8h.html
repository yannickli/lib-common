◆ GENERIC_DELETE()

GENERIC_DELETE ( mem_bench_t ,
mem_bench  )

Declare a mem_bench object to be leaked.

If you cannot ensure the mem_bench object will be wiped before the log module termination (if it must be destroyed by a thr_hook for example), it must be registered with this function. It will do a partial wipe, allowing to work properly.

The mem_bench object must still be mem_bench_wipe'd manually to finish the cleanup.

Wipe time is unspecified, and the allocated memory will not be reclaimed.

Initialize mem_bench object.

The mem_bench object is initialized to dump into a file each period iterations. The filename is "./mem.[\p name].[pid].[address of \p sp]". "[type].[address of sp] is also used for logger name.

Parameters
[in]typeName of mem_bench object.
[in]periodLogging period. 0 means no logging.
